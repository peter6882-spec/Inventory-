## 1. 系統簡介

本系統分為兩個頁面：

| 頁面 | 網址結尾 | 使用對象 |
|------|---------|---------|
| **員工頁** | `/index.html`（主頁） | 日常操作員工 |
| **管理後台** | `/admin2.html` | 管理人員（需 PIN 碼） |

**員工頁功能**：庫存操作（入庫／備料／消耗／回放／報廢）、簽收入庫、銷售回報、盤點。
**管理後台功能**：品項管理、分類管理、叫貨管理、銷售看板、報表中心、系統設定（含公告）。

> 系統資料儲存在雲端（Firebase），所有裝置即時同步。畫面右上角有同步狀態指示（綠燈＝已同步）。

## 2. 安裝到手機桌面

安裝後可像 App 一樣從桌面開啟，不需每次打網址。

### iPhone / iPad（Safari）
1. 用 **Safari** 打開員工頁網址
2. 點下方工具列的**分享按鈕**（方塊加箭頭）
3. 選「**加入主畫面**」→「**新增**」

### Android（Chrome）
1. 用 **Chrome** 打開員工頁網址
2. 點右上角**三個點**
3. 選「**新增到主畫面**」或「**安裝應用程式**」

> 管理後台沒有從員工頁的連結入口，請直接把 `admin2.html` 網址加入書籤，建議只安裝在管理人員的裝置上。
> 前台建議固定使用單一分頁，避免同時開多個分頁造成載入延遲。

## 3. 員工頁操作說明

員工頁上方有頁籤：**庫存操作 ／ 簽收入庫 ／ 銷售回報 ／ 盤點**。右上角可切換中文／越南語。
頁籤下方若有黃色**公告橫幅（大聲公）**，是管理者發布的提醒，請留意。

---

### 3-1. 庫存操作

顯示所有**啟用中**品項，依分類排列。上方有分類篩選按鈕與搜尋框。

**倉位說明**：

| 倉位 | 說明 |
|------|------|
| 大倉冷凍／冷藏／常溫 | 主要儲存區（依品項溫層） |
| 解凍冷藏 | 備料解凍中的區域 |
| 其他備貨 | 其他已備料存放區 |

存量低於安全線會標紅色「補貨」警示，頁面頂端也會列出叫貨提醒。

**操作動作**（桌機直接在該列輸入數量後點動作；手機點卡片展開面板）：

| 動作 | 說明 |
|------|------|
| **入庫** | 手動進貨入庫（一般進貨改由「簽收入庫」處理，此按鈕可由後台隱藏） |
| **備料** | 從大倉移到備料區（需品項開啟「允許備料」） |
| **消耗** | 實際使用時扣庫存 |
| **回放** | 備料沒用完退回大倉（需開啟「允許回放」；限完整未開封品項） |
| **報廢** | 過期或損壞的損耗登記，會跳出原因快選（過期／損壞／其他） |

**單位切換**：若品項有設定基準單位（例：紙巾 1 箱＝20 卷），操作時可切換用「箱」或「卷」輸入，系統自動換算。

**品項明細**：點品項的「明細」可看近 7 天操作記錄。「說明」可看該品項的倉位動線。

---

### 3-2. 簽收入庫

顯示管理者已下單、等待到貨的叫貨單（頁籤會顯示待簽收筆數）。

**流程**：
1. 貨到後，在對應叫貨單填入**實際到貨數量**
2. 數量與叫貨量不符時，會要求選擇**差異原因**（廠商漏送／延後補送／多送／秤重誤差等）
3. 按簽收：
   - **不需處理的品項** → 直接入庫完成
   - **需處理的品項**（如生豬肉）→ 先簽收轉「處理中」，處理完成後再登記入庫數量
4. **短交**（到貨少於叫貨）→ 單子留在進行中，補送到貨可再簽收；**溢收**（多於叫貨）→ 入庫後由管理者結案

> 簽收送出後按鈕會鎖定，避免重複點擊造成重複入庫。

---

### 3-3. 銷售回報

依管理者設定的銷售品項（例：9吋厚、12吋薄）輸入當日銷售數量後送出。

- 同一天可**回報多筆**（例如分班回報）
- 下方「今日回報記錄」可看當天每一筆，當日資料可**修改或刪除**
- 隔天的記錄自動鎖定，不可再改

---

### 3-4. 盤點

1. 點「**盤點**」頁籤，輸入**盤點 PIN 碼**
2. 頁面列出品項各倉位的目前系統庫存，可用搜尋框快速找品項
3. 填入實際清點數量後按「**✅ 確認**」，系統自動調整庫存並記錄差異
4. 每個品項獨立確認，可分批完成；已確認顯示淺綠底

## 4. 管理後台操作說明

**網址**：主頁網址後加 `admin2.html`，登入需輸入**管理 PIN 碼**。
頁籤：**品項管理 ／ 分類管理 ／ 叫貨管理 ／ 銷售看板 ／ 報表中心 ／ 系統設定**。

---

### 4-1. 品項管理

顯示所有品項（含停用），可搜尋。表格含「總庫存」欄（有效庫存，低於安全線標紅）與「倉位（入庫／消耗）」欄。

#### 新增／編輯品項

編輯視窗分四區：

**基本資料**：名稱、分類、越南語名稱、狀態、排序。

**單位與安全線**：
| 欄位 | 說明 |
|------|------|
| 顯示單位 | 員工看到的單位（包、桶、顆…） |
| 基準單位 | 內部計算單位（與顯示不同時填，例 g、卷） |
| 換算 | 幾基準單位 = 1 顯示單位（例 800g=1包 填 800） |
| 安全線 | 低於此量顯示補貨警示 |

**倉位與動線**：入庫至（決定溫層）、允許備料＋備料至、允許回放＋回放至。消耗倉由系統自動判斷顯示（未開備料＝入庫倉；開備料＝備料倉）。

**叫貨設定**：
| 欄位 | 說明 |
|------|------|
| 叫貨單位 | 叫貨用單位，可與庫存不同（例：斤、箱） |
| 叫貨換算 | 1 叫貨單位 = ? 基準單位（叫貨單位＝顯示或基準單位時免填） |
| 到貨需處理 | 開啟後簽收不直接入庫，需登記處理數量 |

編輯視窗下方有**即時預覽**，顯示單位換算與員工可用哪些單位操作。

> 修改倉位設定時若該倉仍有庫存，系統會跳警告確認。停用取代刪除較安全（刪除無法復原）。

---

### 4-2. 分類管理

- 新增／刪除分類（有品項使用的分類不可刪）
- 越南語對照：設定各分類的越南語顯示名稱

---

### 4-3. 叫貨管理

黃色面板列出**低庫存品項**，每個可「＋叫貨」快速建單。

**建立叫貨單**：選品項、填叫貨量與單位，可「先建立稍後下單」或「建立並確認下單」。

**狀態流程**：待下單 → 已下單（員工可簽收）→ 部分到貨／溢收（待結案）→ 完成。

- **進行中**列表：可確認下單、取消、結案；顯示叫貨量／簽收進度與差異記錄
- **已完成**列表：可載入更多、匯出 Excel、清除全部紀錄（雙重確認）

---

### 4-4. 銷售看板

- 上方**今日合計卡片**：各銷售品項當日總量，即時更新
- **日彙總表**：近 30 天，同日多筆自動加總，點「筆數」可展開當日每筆明細
- 工具列：**匯出 Excel**、**匯出 Google Sheet**（需先設定端點，見系統設定）、**品項設定**（新增／改名／刪除銷售品項、設定 Google Sheet 端點）

---

### 4-5. 報表中心

上方可切換報表類型，共用下方的日期區間（今天／昨天／本週／本月／自訂）與分類篩選，按「查詢」產生，各類型皆可匯出 Excel。

| 報表 | 內容 |
|------|------|
| **庫存異動** | 各品項期初／入庫／消耗／回放／報廢／盤點調整／期末 |
| **消耗排行** | 消耗量／次數、報廢量、報廢率 |
| **報廢統計** | 報廢量／次數、原因分解、報廢率 |
| **盤點差異** | 盤點次數、盤盈／盤虧／淨調整 |
| **叫貨分析** | 叫貨次數、總量、簽收、短交／溢收、平均完成天數 |
| **異動明細** | 期間內每筆入庫／備料／消耗／回放／報廢操作流水（時間＋內容） |

> 分析報表的欄位可點標題排序。備料只是倉位間移動，不計入消耗。

### 4-6. 系統設定

左側分頁：

#### PIN 碼管理
- **管理後台 PIN**：登入後台用，預設 `1234`，請務必修改
- **盤點 PIN**：員工進入盤點用，可獨立設定或清除

#### 品項工具
- **匯出備份**：所有品項設定匯出為 JSON（建議定期備份；不含庫存數量）
- **批次匯入**：可貼上 JSON 或**上傳 JSON 檔**（如備份檔），同名品項自動跳過、庫存一律歸零。也可用 CSV 範本＋Gemini AI 產生 JSON

#### 越南語翻譯
用 Gemini 產生品項／分類的越南語翻譯後匯入。

#### 公告設定
- 輸入公告內容、選跑馬燈或靜態橫幅樣式
- 可設**上稿／下稿時間**排程（留空＝立即並持續），到點自動顯示／隱藏
- 有即時預覽；「清除並停用」可撤下

#### 系統維護
- **前台功能**：可開關前台的「入庫」按鈕（一般進貨走簽收入庫，建議關閉避免誤觸）
- **明細日誌索引（首次設定）**：首次用品項明細需在 Firebase 建一次索引，依畫面步驟操作
- **操作記錄管理**：查詢筆數、清理 6 個月前記錄
- **危險區域**：一鍵清除所有庫存與記錄（雙重確認、無法復原）

## 5. 常見問題

**Q：畫面一直卡在「正在載入資料庫」**
A：多半是同裝置開了多個分頁互相搶快取。把整個瀏覽器完全關閉再重開即可。平常前台固定用單一分頁。

**Q：右上角同步狀態顯示「同步中」很久**
A：那代表背景還在跟雲端核對，畫面其實可以正常操作——以每次操作後的成功提示為準即可，不必等該狀態。真的連線失敗會顯示紅燈「連線失敗」。

**Q：出現紅色「目前離線」橫幅**
A：網路斷線，操作無法儲存。恢復網路後橫幅自動消失。

**Q：點操作按鈕沒反應**
A：確認是否離線；有網路仍無反應就重新整理頁面。

**Q：改了品項倉位／單位，員工頁沒更新**
A：前台重新整理即可；若加到主畫面的 PWA 沒更新，多重新整理一兩次讓新版生效。

**Q：庫存數字感覺不對**
A：到「報表中心 → 異動明細」看該期間做了哪些操作；確認要修正就用「盤點」把庫存調成實際數量。

**Q：忘記 PIN 碼**
A：請聯絡系統管理員；完全無法登入需請工程師到 Firebase 後台查設定值。

**Q：如何讓新員工使用**
A：把員工頁網址傳給員工，請用 Safari（iPhone）或 Chrome（Android）開啟並加入主畫面；需要盤點時提供盤點 PIN 碼。
