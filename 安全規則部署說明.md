# Firestore 安全規則部署說明

規則檔：專案根目錄的 `firestore.rules`。
發布只需兩分鐘，照以下步驟操作。

## 部署步驟

1. 開啟 [Firebase 控制台](https://console.firebase.google.com/)，選擇專案 **caishushu-inventory**
2. 左側選單 → **建構** → **Firestore Database**
3. 上方切到 **規則（Rules）** 分頁
4. 全選刪除舊規則，把 `firestore.rules` 的**整份內容**貼上
5. 按 **發布（Publish）**，等幾秒生效

## 發布後驗證（重要，花 3 分鐘走一遍）

規則若有誤會直接擋住正常功能，發布後請立刻測試：

| 測試 | 預期結果 |
|------|----------|
| 前台：入庫或消耗一筆 | ✅ 正常寫入 |
| 前台：銷售回報送出一筆、修改、刪除 | ✅ 正常 |
| 前台：盤點修改一個品項 | ✅ 正常 |
| 後台：編輯品項儲存 | ✅ 正常 |
| 後台：建叫貨單 → 前台簽收 | ✅ 正常 |
| 後台：修改分類、修改 PIN | ✅ 正常 |

若某個操作跳出「寫入失敗 / permission」錯誤，代表規則與該功能衝突：
先回控制台把規則暫時改回 `allow read, write: if true;` 恢復營運，再回報錯誤訊息。

## 這份規則保護了什麼

- **路徑白名單**：只有系統用到的 5 個位置能讀寫，其他路徑全部拒絕（外人無法在你的資料庫裡開新集合塞資料）
- **禁止刪除核心文件**：庫存主檔與設定檔不可能被整份刪掉
- **欄位白名單＋型別檢查**：寫入的資料必須符合系統格式，亂塞欄位或錯誤型別會被拒絕
- **日誌不可竄改**：操作記錄只能新增與清除，不能事後修改內容

## 已知限制（誠實告知）

系統目前沒有登入機制（只有前端 PIN），因此**知道 Firebase 專案設定的人，
仍可用符合格式的資料讀寫資料庫**。這份規則擋的是亂寫、誤刪與外部濫用，
不是完整的存取控制。若未來需要更強的保護（例如帳號登入、角色權限），
需加入 Firebase Authentication，屆時再規劃。
